FluoRender Debian Packaging Instructions (Ubuntu)

------------------------------------------------------------
1. Install Docker (one-time setup)
------------------------------------------------------------

sudo apt update
sudo apt install -y docker.io

Enable and start Docker:

sudo systemctl enable docker
sudo systemctl start docker

Allow running Docker without sudo:

sudo usermod -aG docker $USER

Apply group change:

newgrp docker

Test Docker:

docker run hello-world


------------------------------------------------------------
2. Prepare packaging directory
------------------------------------------------------------

Inside fluorender/Install/Ubuntu, create:

fluorender/
├── debian/
│   ├── control
│   ├── rules
│   └── install
├── build/
│   └── bin/
├── assets/
│   ├── fluorender.desktop
│   └── icons/
│       └── hicolor/
│           ├── 48x48/apps/fluorender.png
│           └── 128x128/apps/fluorender.png
├── build_and_test.sh

NOTES:
- Icons MUST be named exactly: fluorender.png
- Do NOT use filenames like fluorender_48.png


------------------------------------------------------------
3. Copy build output (binary + resources)
------------------------------------------------------------

mkdir -p fluorender/build/bin
rm -rf fluorender/build/bin/*
cp -r ../../build/bin/* fluorender/build/bin/

IMPORTANT:
- Copy EVERYTHING under build/bin (not just FluoRender)
- Required for Scripts, CL_code, Templates, etc.


------------------------------------------------------------
4. Set permissions
------------------------------------------------------------

chmod +x fluorender/debian/rules
chmod 755 fluorender/build/bin/FluoRender


------------------------------------------------------------
5. Configure debian/install (CRITICAL)
------------------------------------------------------------

Use EXACTLY:

build/bin/FluoRender usr/bin/
build/bin/* usr/lib/fluorender/

assets/icons/hicolor/* usr/share/icons/hicolor/
assets/fluorender.desktop usr/share/applications/


IMPORTANT:
- Use "hicolor/*", NOT "hicolor/" (avoids nested folders)
- Do NOT rename files during install


------------------------------------------------------------
6. Build and test package (Docker)
------------------------------------------------------------

cd fluorender/Install/Ubuntu/fluorender
./build_and_test.sh

This script will:

- Use a clean Ubuntu container each run
- Install dependencies
- Build the package
- Install and test the package
- Validate shared library dependencies

Docker ensures reproducible results (no leftover system state).


------------------------------------------------------------
7. Clean local system before testing (IMPORTANT)
------------------------------------------------------------

sudo apt remove --purge -y fluorender || true

sudo rm -rf /usr/share/icons/hicolor/*/apps/fluorender*
sudo rm -f /usr/share/applications/fluorender.desktop

# optional (recommended for debugging)
rm -rf ~/.config/FluoRender


------------------------------------------------------------
8. Locate generated package
------------------------------------------------------------

ls build-area/*.deb

You will see:

fluorender_<version>_amd64.deb      ← main package (use this)
fluorender-dbgsym_<version>*.ddeb  ← debug package (ignore)


------------------------------------------------------------
9. Fix file ownership (remove lock icon)
------------------------------------------------------------

sudo chown -R $USER:$USER build-area


------------------------------------------------------------
10. Install locally
------------------------------------------------------------

sudo apt install ./build-area/fluorender_*.deb


------------------------------------------------------------
11. Verify installation (recommended)
------------------------------------------------------------

# Check icons
ls /usr/share/icons/hicolor/*/apps/fluorender.png

# Check desktop entry
ls /usr/share/applications/fluorender.desktop

# Check resources
ls /usr/lib/fluorender


------------------------------------------------------------
12. Refresh desktop environment
------------------------------------------------------------

gtk-update-icon-cache /usr/share/icons/hicolor
update-desktop-database

# Then log out and log back in


------------------------------------------------------------
13. Run application
------------------------------------------------------------

FluoRender

or:

gtk-launch fluorender


------------------------------------------------------------
14. Release on GitHub
------------------------------------------------------------

Use GitHub Releases (NOT git commit):

1. Repository → Releases → "Draft a new release"
2. Set version tag (e.g., v2.34.1)
3. Upload:

   fluorender_<version>_amd64.deb

Do NOT upload:

   *.ddeb


------------------------------------------------------------
15. Notes (Important)
------------------------------------------------------------

- Package size (~5–50 MB) is normal:
  system libraries are installed separately via apt.

- Target:
  Ubuntu 22.04 (Jammy)

- Icons REQUIRE:
  correct filename (fluorender.png)
  correct directory (hicolor/*/apps)

- Resource handling:
  Use GetResourcePath() in code for fallback logic

- User config:
  Stored in ~/.config/FluoRender

- Common failure cause:
  leftover files from previous installs
